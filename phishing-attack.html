Overview


Phishing is a common type of cyber attack that targets individuals through email, text messages, phone calls, and other forms of communication. A phishing attack aims to trick the recipient into falling for the attacker's desired action, such as revealing financial information, system login credentials, or other sensitive information.

1. Phishing Techniques

Cybercriminals use three primary phishing techniques to steal information: malicious web links, malicious attachments, and fraudulent data-entry forms.

Malicious Web Links

Phishing links take users to impostor websites or sites infected with malicious software. These links are often disguised as trusted URLs and embedded in logos or images.

Example of Link Manipulation

Original URL: https://www.paypal.com/login
Phishing URL: https://www.paypa1.com/login
                    
Hidden Link HTML:
https://www.paypal.com/login

Malicious Attachments

These attachments appear legitimate but contain malware that can compromise computers and their files.

Common Malicious Attachment Types

  • .exe - Executable files
  • .doc or .docm - Macro-enabled documents
  • .pdf - PDF files with embedded scripts
  • .zip - Compressed files hiding malware

Fraudulent Data Entry Forms

Fake forms that prompt users to enter sensitive information like passwords, credit card data, and personal details.

Protection Tips:

  • Always verify the sender's email address
  • Check URLs carefully before clicking
  • Never open unexpected attachments
  • Use anti-phishing browser extensions

2. Advanced Phishing Types

Phishing has evolved into various sophisticated forms targeting different aspects of digital communication and specific groups of users.

Email Phishing

The general term given to any malicious email message meant to trick users into divulging private information. Attackers generally aim to steal account credentials, personally identifiable information (PII) and corporate trade secrets.

Common Email Phishing Indicators

  • Generic greetings instead of personalized ones
  • Spelling and grammar errors
  • Urgency or threatening language
  • Suspicious sender email domains

Spear Phishing

Targeted attacks aimed at specific individuals or organizations, often using personalized information to appear more credible.

Spear Phishing Example

From: ceo@company-name.com
To: finance@company-name.com
Subject: Urgent Wire Transfer Needed

Hi Sarah,

I'm in a meeting but need you to process an urgent wire transfer.
Amount: $28,500
Beneficiary: ABC Consulting Ltd
Account: 1234567890

Please handle this ASAP and send confirmation.

Best regards,
John Smith
CEO

Link Manipulation

Messages contain a link to a malicious site that looks like the official business but takes recipients to an attacker-controlled server where they are persuaded to authenticate into a spoofed login page that sends credentials to an attacker.

Link Manipulation Techniques

1. Lookalike domains: bankofamerica-secure.com vs. bankofamerica.com
2. IDN homograph attacks: using characters from different languages
   Example: аррӏе.com (using Cyrillic 'а' and 'р')
3. URL shorteners: hiding malicious URLs behind bit.ly, tinyurl.com, etc.
4. HTML tricks: https://legitimate-bank.com
5. Typosquatting: misspelled domains like amazom.com or gooogle.com

Whaling & CEO Fraud

High-stakes attacks targeting senior executives or using executive impersonation to authorize fraudulent transactions.

CEO Fraud Analysis

Attack Pattern:
1. Research target company structure
2. Create lookalike domain (ceo@company-name.co vs .com)
3. Impersonate CEO's writing style
4. Request urgent financial action
5. Pressure recipient with time constraints
6. Emphasize confidentiality

Smishing & Vishing

SMS-based (Smishing) and voice-based (Vishing) phishing attacks that exploit mobile and telephone communications.

Common Attack Patterns

Smishing Example:
"Your account has been suspended. Click here to reactivate: bit.ly/scam-link"

Vishing Script:
"Hello, this is Microsoft Tech Support. We've detected a virus on your computer..."

Content Injection

An attacker who can inject malicious content into an official site will trick users into accessing the site to show them a malicious popup or redirect them to a phishing website.

Example Attack Scenario

1. Attacker exploits XSS vulnerability on legitimate website
2. Injects script: <script>window.location='https://fake-login.com'</script>
3. When users visit the compromised page, they're redirected to phishing site
4. Users believe they're still on the legitimate website
5. Enter credentials, which are sent to the attacker

Malware-Based Phishing

Users tricked into clicking a link or opening an attachment might download malware onto their devices. Ransomware, rootkits, or keyloggers are common malware attachments that steal data and extort payments from targeted victims.

Common Malware Delivery Methods

  • Macro-enabled documents (.docm, .xlsm)
  • Executable files disguised as PDFs or images
  • JavaScript attachments (.js)
  • Archive files with hidden executables (.zip, .rar)

"Evil Twin" Wi-Fi

Spoofing free Wi-Fi, attackers trick users into connecting to a malicious hotspot to perform man-in-the-middle exploits.

Attack Methodology

1. Attacker creates wireless access point with name similar to legitimate network
   Example: "Starbucks_Free_WiFi" vs. official "Starbucks WiFi"
2. Positions themselves in popular location (coffee shop, airport, hotel)
3. When victims connect, all traffic passes through attacker's device
4. Attacker can:
   - Monitor all unencrypted traffic
   - Inject malicious code into websites
   - Redirect to phishing pages
   - Capture login credentials

Pharming

Pharming is a two-phase attack used to steal account credentials. The first phase installs malware on a targeted victim and redirects them to a browser and a spoofed website where they are tricked into divulging credentials. DNS poisoning is also used to redirect users to spoofed domains.

DNS Cache Poisoning Attack

Normal DNS Resolution:
1. User types "bank.com" in browser
2. Computer checks local DNS cache
3. If not found, query sent to ISP's DNS server
4. DNS server returns IP address for bank.com
5. Browser connects to correct website

Pharming Attack:
1. Attacker compromises DNS server or poisons DNS cache
2. Modified DNS records point bank.com to attacker's IP
3. User types "bank.com" but is sent to fake site
4. URL shows correct domain, but site is controlled by attacker

Angler Phishing

Using social media, attackers reply to posts pretending to be an official organization and trick users into divulging account credentials and personal information.

Social Media Attack Vector

1. User posts complaint about service on company's social media page
   Example: "@AirlineX your flight was delayed and I missed my connection!"
2. Attacker creates account resembling support: "@AirlineX_Support"
3. Attacker responds: "We apologize for the inconvenience. Please verify your booking at https://airlinex-refund.com to process compensation"
4. User follows link, enters personal and payment information
5. Attacker harvests data for fraud or identity theft

Watering Hole

A compromised site provides endless opportunities, so an attacker identifies a site used by numerous targeted users, exploits a vulnerability on the site, and uses it to trick users into downloading malware. With malware installed on targeted user machines, an attacker can redirect users to spoofed websites or deliver a payload to the local network to steal data.

Targeted Watering Hole Attack

Attack Process:
1. Identify websites frequently visited by target organization
   Examples: Industry forums, supplier portals, partner websites
2. Compromise selected website through vulnerability exploitation
3. Inject malicious code that targets specific organizations by IP ranges
4. When target employees visit the site, malware is delivered silently
5. Establish persistence in the target network through installed backdoors
6. Exfiltrate sensitive data or establish command & control

Protection Tips:

  • Implement multi-factor authentication
  • Verify unusual requests through alternative channels
  • Train employees to recognize social engineering
  • Use DMARC email authentication

3. Prevention & Security

Effective phishing prevention requires a multi-layered approach combining user training, technical controls, and organizational policies.

User Training & Awareness

Training users to detect a phishing email is crucial. A sense of urgency and requests for personal data, including passwords, embedded links and attachments, are all warning signs. Users must be able to identify these warning signs to defend against phishing.

Key Training Elements

  • Regular phishing simulations
  • Security awareness workshops
  • How to identify suspicious emails
  • Reporting procedures for suspected phishing

Email Security & Filters

Use anti-phishing email security systems. Artificial intelligence scans incoming messages, detects suspicious messages and quarantines them without allowing phishing messages to reach the recipient's inbox.

Email Security Configuration

admin@server:~$ cat email_security.conf
# SPF Record
v=spf1 ip4:192.0.2.0/24 include:_spf.google.com ~all

# DKIM Configuration
selector._domainkey IN TXT "v=DKIM1; k=rsa; p=MIGfMA0..."

# DMARC Policy
_dmarc IN TXT "v=DMARC1; p=reject; rua=mailto:reports..."

Safe Browsing Practices

Avoid clicking links directly from emails. Instead of clicking a link and authenticating into a web page directly from an embedded link, type the official domain into a browser and authenticate directly from the manually typed site.

URL Verification Technique

1. Received email claiming to be from your bank
2. INCORRECT: Clicking on "Click here to verify your account"
3. CORRECT: Open a new browser tab
4. Type your bank's official URL manually (e.g., www.mybank.com)
5. Navigate to the relevant section through official site menus
6. Log in with your credentials only on the official site

Password Management

Change passwords regularly: users should be forced to change their passwords every 30-45 days to reduce an attacker's window of opportunity. Leaving passwords active for too long gives an attacker indefinite access to a compromised account.

Password Best Practices

  • Use a different password for each account
  • Create complex passwords (12+ characters with mixed case, numbers, symbols)
  • Consider using a password manager
  • Enable multi-factor authentication wherever possible

System & Software Updates

Keep software and firmware up-to-date: software and firmware developers release updates to remediate bugs and security issues. Always install these updates to ensure known vulnerabilities are no longer present in your infrastructure.

Network Security Controls

Install firewalls: firewalls control inbound and outbound traffic. Malware installed from phishing silently eavesdrops and sends private data to an attacker, but a firewall blocks malicious outgoing requests and logs them for further review.

Basic Firewall Configuration

admin@server:~$ cat /etc/iptables/rules.v4
# Default policies
-P INPUT DROP
-P FORWARD DROP
-P OUTPUT ACCEPT

# Allow established connections
-A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT

# Allow SSH, HTTP, HTTPS
-A INPUT -p tcp --dport 22 -j ACCEPT
-A INPUT -p tcp --dport 80 -j ACCEPT
-A INPUT -p tcp --dport 443 -j ACCEPT

# Block outgoing connections to known malicious domains
-A OUTPUT -d malicious-site.com -j DROP
-A OUTPUT -d phishing-domain.net -j DROP

Browser & Popup Security

Avoid clicking on popups: attackers change the location of the X button on a popup window to trick users into opening a malicious site or downloading malware. Popup blockers stop many popups, but false negatives are still possible.

Browser Security Settings

  • Enable popup blockers in your browser
  • Install security extensions (uBlock Origin, Privacy Badger)
  • Use browser phishing protection features
  • When encountering a popup, use Alt+F4 instead of clicking X

Financial Information Protection

Be cautious about giving out credit card data: unless you know the site is completely trustworthy, never give credit card data to a website you don't recognize. Any site promising gifts or money back should be used with caution.

Safe Online Shopping Practices

  • Verify HTTPS/SSL is active (lock icon in browser)
  • Use credit cards instead of debit cards for purchases
  • Consider virtual credit cards for one-time transactions
  • Check website reviews and reputation before purchasing
  • Use trusted payment processors (PayPal, Apple Pay, Google Pay)

Incident Response

Procedures for handling successful phishing attacks and preventing future incidents.

Incident Response Plan

1. IDENTIFICATION
- Detect phishing incident
- Document initial findings
- Alert security team

2. CONTAINMENT
- Isolate affected systems
- Block malicious URLs/IPs
- Disable compromised accounts

3. ERADICATION
- Remove malware
- Reset credentials
- Patch vulnerabilities

4. RECOVERY
- Restore from backups
- Verify system integrity
- Resume operations

5. LESSONS LEARNED
- Update security controls
- Enhance training
- Improve procedures

Protection Tips:

  • Deploy email authentication protocols (SPF, DKIM, DMARC)
  • Implement AI-based phishing detection
  • Conduct regular security assessments
  • Maintain incident response readiness
